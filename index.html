requireJs

Posted on 2017-09-16 | In 技术 , 插件及框架的使用

一、使用模块化的好处

1. 具有实现一个完整功能的整体
2. 使用模块化的好处
    + 为了更好地复用
    + 减少全局变量污染
    + 解决功能之间的依赖关系
3.requireJS/angular/vue都是适用于：单页应用：Single Page Application(SPA)

二、使用闭包来实现模块化

+ 闭包实现存在的缺点
    - 随着功能的增加，暴露的全局变量也会越来越多
        - requireJS几乎没有全局变量
    - 不能很好的解决模块依赖的问题

三、AMD规范和CMD规范比较大的区别：

1.AMD规范就是指requireJS
    -->async module define：异步模块定义(异步加载模块、依赖前置)
        -->模块定义和导入(使用)：define定义一个模块，define(["导入的模块"])/require(["导入模块"])
        -->异步加载模块   动态创建script标签
        -->依赖前置：把所有依赖的模块放在文件的头部

2.CMD规范就是指的seajs
    -->common module define：通用模块定义(同步加载模块)
        -->模块定义和导入：exports/module.exports
        -->同步加载模块
        -->依赖就近：50行代码使用商品模块的功能，就在49行依赖商品模块
            -->49行：var product=require("product");  //导入商品模块
            -->50行：product.add([]);                 //初始化一个商品
CommonJS规范：nodeJS

四、入门

1.  引入requirejs文件

2. 编写js代码
   在a.js文件中通过来定义一个模块(tip:约定一个js文件就是一个模块)

        define([],function(){
          //模块内部的代码
        })

   在b.js文件中从而获取a.js中的代码，这样的话就可以保证在执行b.js中的代码之前先执行了a.js中的代码

        require(["a"],function(){

        })

加载了require.js，并且我们在加载require.js这段代码的后面通过require函数导入了

五、依赖注入：模块具有返回值

1. 入口文件
    在index.html中引入requirejs时的script标签添加data-main属性值，那么requirejs就会自动加载该文件(模块)，可以通过该文件来进行访问其他的模块(这里引入main.js文件)

         <script src="require.js" data-main="./main"></script>

2. 给定义模块的回调函数添加return语句
    //a.js文件中
    define(function(){
        //模块的逻辑
        return function(){
            console.log('a文件加载成功')
        }
    })

    //b.js文件中
    define(function(){
        return function(){
            console.log('b文件加载成功')
        }
    })

    //c.js文件中
    define(function(){
       return function(){
            console.log('c文件加载成功')
        }
    })


3.创建一个main.js文件，load.js依赖了三个模块a.js,b.js,c.js,在获取模块的时候，通过require()的第二个参数来依次获取各个模块的返回值
    //main.js文件中
    require(["./a.js","./b.js","./c.js"],function(a,b,c){
        a();
        b();
        c();
      })

4.通过依赖注入可以实现按需加载

    requireJs使用的是异步加载，所以我们加载多个模块的时候就会发生加载顺序不正确的问题，main.js依赖了3个子模块，这3个子模块加载顺序，我们是不能决定的。我们能决定的是：3个子模块功能的执行顺序
5.关于依赖注入 子模块功能的执行顺序的问题

    实际开发中，可能一个js文件会依赖无数个模块，对于无数个模块有的有返回值，有的没有返回值，在添加模块依赖的时候，应该注意：把所有的有返回值的模块在前面添加，把没有返回值的模块在后面添加

    例举例：

         //这里a.js和c.js有返回值，b.js无返回值，下面这种写法会导致a模块可以正常获取，c接受了b模块的返回值，b模块没有返回值，所以c的值为：undefined
             require(["./a.js","./b.js","./c.js"],function(a,c){

              })

     所以正确写法应该为：

            require(["./a.js","./c.js","./b.js"],function(a,c){
                    //a接受a模块返回值
                    //c接受c模块返回值
                    //b模块没有返回值，就不用接收
              })

六、引入第三方文件

1.对于任何第三方模块来说，都可以在requirejs中引用该模块的路径从而使用该模块的功能    
    支持AMD规范的模块（有返回值,可以使用形参接收）：
        require(["jquery-2.1.2","template-web"], function ($,template){

        }))
    不支持MD规范的模块,但是它又依赖于其他模块(比如jquery)，就可以通过shim来解决，下面将会作出解释；

2.配置常用的模块路径(比如：jquery/arttemplate)
    1. require.config({
        baseUrl:"./lib",
        paths:{
            //路径不需要.js后缀，最终该文件的路径将会和baseUrl拼接，也就是："./lib/js/jquery-2.1.4"
            jquery:"js/jquery-2.1.4"
            bootstrap:"assets/bootstrap/bootstrap"
        }，
         shim:{
            //这里的bootstrap就是上面的paths中的bootstrap
            bootstrap:{
                //这样在请求bootstrap的时候会自动去请求jquery模块
                deps:["jquery"] //这里的"jquery"也是上面paths中的jquery
            }
        }
    })
    //a.js文件中
    define(["jquery"],function($){
        //这里请求的jquery也就是去上面paths中设置的jquery进行查找,也就是："./lib/js/jquery-2.1.4"

    })
3.shim只用于第三方模块
    shim一般适用于：引入了2个第三方模块，之间存在依赖，其中一个不支持AMD规范导入了jquery/bootstrap，bootstrap依赖了jquery，因为bootstrap不支持AMD规范，所以采用了shim

4.如果引用了2个第三方模块，2个第三方模块都支持AMD规范，那么就可以使用AMD规范中的写法

5.检测第三方库是否支持AMD规范？
    查看源代码，看看是否有以下内容？如果有，说明支持AMD规范，如果没有说明不支持

    if (typeof define === "function" && define.amd) {}

一、JQ入口函数根据不同参数创建实例

### 一、 创建实例的样式
    1. 传入null、undefined、0、NaN、''： 统一返回空的实例；

    3. 如果传入的是html片段，那么会根据这些片段创建DOM对象，然后按照下标存储起来，同时添加一个length属性代表数量；
    4. 如果传入的是数组，那么会把数组中的每一项分别添加到实例身上，同时把length也加过来；
    5. 如果传入的是伪数组，那么会把数组中的每一项分别添加到实例身上，同时把length也加过来。
        1. 伪数组的特点：
            1. 必须要有length属性
            2. 如果length属性值为0，那么这个对象有没有元素无所谓
            3. 如果length属性值不为0，那么这个对象一定有 (length - 1) 为下标的属性值。
            4. var obj = { length: 10, 0: 'a', 1: 'b', 2: 'c', 3: 'd', 4: 'e', 5: 'f' };
    6. 如果传入的是对象，那么直接把这个对象按照下标添加到实例身上，同时添加一个length属性代表数量;
    7. 如果传入的是基本数据类型，null、undefined除外，字符串除外，那么直接把这个数据添加到实例身上，同时添加一个length属性代表数量。
    8. 如果传入的是DOM，那么把这个DOM添加到实例身上，同时添加一个length属性代表数量。

### 二、 实例对象

    1. jQuery实例对象是一个伪数组对象
    2. jquery对外暴露的jquer和$ ，这两方法实际上是同一个方法，调用这两个方法，可以得到jQuery实例对象；
    3. jquer和$ 实际上是一个工厂函数；
    4. 验证实例是伪装组;
        1. console.log(({}).toString.call($div));

二、jQuery插件写法

1. jQuery.extend
       这种功能一般是为$对象 直接增加可以访问的方法  使用频率较低
          $.extend({
                sayHello: function () {
                  console.log('你好我是jQuery,这是一个打招呼的插件');
                  console.log(this);
                  console.log(this == $);
                }
              })
              // this 是$
              $.sayHello();
2. $.fn.extend   this 是jQuery对象
    $.fn.extend({
        becomeYellow: function () {
          console.log('我们是黄种人');
          // $('div').becomeYellow();
          console.log(this);
          this.css('backgroundColor', 'yellow');
          // 为了能够链式编程 还要 返回 this
          return this;
        }
      }) 
3. 特殊情况  这里this不是jquery对象，而是Dom 对象
    $('body').click(function(){
         console.log(this);
         this.innerHTML = '我是Dom';
       })//知识绑定事件，实际点击时是document.body.onclick
      jQuery基于js封装的一些函数库
    $('body').click(function(){console.log(this);});
      内部可以理解为,就是 document.body.onclick  = function(){}
      当我们触发了 body的 点击事件时 等同于 document.body.onclick();
      所以这里的this 就是 绑定的那个 dom对象

三、jquery对象和dom元素

1. 每一次调用$(),都会产生一块新内存；虽然jquery对象不同，但是对象中的元素('input')('body')相同，都指向同一个dom元素,所以事件就绑定在了相同的元素中 
        页面中有一个<input type="text">
        var $1=$("input").on("click", function () {
           alert("第一次");
       });

       var $2=$("input").on("click", function () {
           alert("第二次");
       });
       console.log($1==$2);//false
       console.log($1[0]==$2[0]);//true

       console.log($("body")==$("body"));//false
       console.log($(document.body)!=$(document.body));//false
       console.log($("body")[0]==$("body")[0]);//true
2. jquery对象不同，通过jquery对象包裹的DOM元素也不同，所以事件绑定在2个不同的元素中
    +
       var $1=$("<input type='button'/>").appendTo("body").on("click", function () {
           alert("111");
       });
       var $2=$("<input type='button'/>").appendTo("body").on("click", function () {
           alert("222");
       });
       console.log($1==$2);//false
       console.log($1[0]==$2[0]);//false
    +
       var div1=document.createElement("div");
       var div2=document.createElement("div");
       console.log(div1==div2);//false

3.两种事件绑定写法

    1.错误的事件绑定写法：
        页面中点击按钮，不会触发事件，页面中的按钮相当于：dom2，绑定事件的按钮是dom1  dom1!=dom2；
           var input = "<input type='button'/>";
           $(input).on("click", function () {
               alert("111");
           });
           $(input).appendTo("body");
    2.正确的事件绑定写法
            var input = "<input type='button'/>";
            var $input=$(input);
           $(input).on("click", function () {
               alert("111");
           });
           $(input).appendTo("body");
    3.补充：
        1.jQuery中绑定的事件是不会被覆盖的
        2.  div1.attachEvent可以绑定多个事件，但是会按照时间倒序触发（ie 6-8）
            div1.addEventListener()可以绑定多个事件，但是会按照排列顺序依次触发
            js中onclick绑定多个事件会被覆盖掉

一、cookie

1.cookie值基于会话机制存储的，随着会话的结束(浏览器关闭)，该cookie值就丢失了

2.原生JS
    1.设置cookie
        document.cookie="id=666";
    2.获取当前回话cookie值
        console.log(document.cookie);
    3.cookie也可以长期保存
        document.cookie="age=18;expires="+new Date("2017-09-12 08:00:00");

3.jquery.cookie.js
    1.设置cookie
        $.cookie("gender","男")
    2.获取当前回话cookie值
        console.log($.cookie("gender"));
    3.设置一个长期cookie
        $.cookie("email","abc@qq.com",{
            //expires的值可以是一个Date类型的对象，也可以是一个以天为单位的数字
            expires:0.5
        });
    4.移除一个cookie
         $.removeCookie("gender");
 4.封装cookie函数

     1.函数封装一
